# Phase 2: 需求分析与规划实战

## 概念 1: Explore → Plan → Code → Commit 工作流

### 工作流概览
```
Explore（探索调研）→ Plan（规划方案）→ Code（执行实现）→ Commit（提交保存）
```

### 第一步：Explore（探索调研）
- **目的：** 快速理解现有代码结构，找到需要改动的位置
- **工具：** Agent 工具 + `subagent_type=Explore`
- **特性：**
  - 使用 **Haiku 模型**（最小最快）
  - 只读模式，不会修改文件
  - 自动总结探索结果

```python
Agent({
  description: "探索认证模块结构",
  subagent_type: "Explore",
  prompt: "探索这个项目的认证模块：
  1. 找到登录相关的所有文件
  2. 列出认证流程的关键步骤
  3. 找出 token/JWT 相关的代码位置"
})
```

### 第二步：Plan（规划方案）
- **目的：** 明确要做什么、怎么做、有什么风险，再动手
- **进入方式：**
  1. 快捷键：**Shift+Tab 两次**
  2. 命令：`/enter-plan-mode`
  3. Agent 工具使用 `subagent_type: "Plan"`

### 第三步：Code（执行实现）
- 基于规划好的方案进行代码实现
- 每个步骤完成后验证
- 有新问题就回到 Plan

### 第四步：Commit（提交保存）
- 使用约定式提交格式：
  - `feat:` 新功能
  - `fix:` Bug 修复
  - `refactor:` 重构
  - `docs:` 文档
  - `test:` 测试
  - `chore:` 杂项

### 为什么要分离 Explore 和 Code？
- 边想边改，思路不清 → 先想清楚，再动手
- 改错了才回头，浪费 → 规划阶段发现风险
- 难以判断完成度 → 规划 = 检查清单

## 概念 2: 需求拆解与技术方案设计

### 需求拆解的核心要素

| 要素 | 作用 | 示例 |
|------|------|------|
| **需求描述** | 要实现什么 | "用户可以上传头像" |
| **约束条件** | 限制条件 | "文件最大 2MB，只支持 JPG/PNG" |
| **参考模式** | 参照现有代码 | "参考现有的 image-upload 组件" |
| **示例文件** | 具体的代码示例 | "看下 src/components/FileUpload.vue" |

**信息越充分，输出越精准。**

### 规划输出应该包含什么？

1. **需求理解** — 对需求的准确描述
2. **变更文件清单** — 具体要改哪些文件
3. **实现步骤** — 按顺序的执行清单
4. **风险评估** — 可能的坑和应对方案
5. **测试策略** — 如何验证

### 迭代优化
- **确认方案：** "这个方案可以，开始写代码"
- **要求调整：** "我觉得第三步可以简化"
- **补充信息：** "再考虑一个边界情况：..."

## 概念 3: 架构规划与决策记录（ADR）

### 什么是 ADR？
ADR = Architecture Decision Record（架构决策记录）

记录：
- 面临什么技术决策
- 为什么选这个方案
- 有什么替代方案，为什么没选

### ADR 模板
```markdown
# ADR-001: 使用 Redis 作为 Session 存储

## 状态
已通过

## 背景
项目需要支持分布式部署，当前使用内存存储 Session 会导致多实例部署时 Session 不共享

## 决策
使用 Redis 存储 Session

## 理由
1. Redis 已经是项目基础设施
2. 支持过期自动清理
3. 性能足够满足需求
4. 团队熟悉 Redis

## 替代方案
1. **JWT 无状态** → 缺点：无强制登出能力
2. **Memcached** → 缺点：功能不如 Redis 丰富
3. **数据库存储** → 缺点：性能不如 Redis

## 影响
- 需要增加 redis-session 依赖
- 部署时需确保 Redis 可用

## 决定人
@张三

## 日期
2026-05-08
```

### ADR 的价值
- 记录决策理由，防止"忘了为什么选这个"
- 未来维护者理解为什么这样设计
- Code Review 时可以对照检查

## 概念 4: 使用 Claude 理解新代码库

### 核心工具：Explore 子智能体

```python
Agent({
  description: "探索项目认证流程",
  subagent_type: "Explore",
  prompt: "这个项目的认证流程是怎样的？
  1. 找到登录相关的所有文件
  2. 列出认证流程的关键步骤
  3. JWT/token 存放在哪里"
})
```

### 正确的提问方式
| ❌ 模糊提问 | ✅ 精准提问 |
|------------|------------|
| "这个项目怎么用？" | "项目用什么框架？入口文件在哪？" |
| "认证怎么做的？" | "登录接口是哪个文件？token 怎么验证？" |

### 交叉验证
Claude 说的不一定全对，用 Grep/Glob 交叉验证

### 生成代码地图
让 Claude 生成项目结构文档，保存到 `docs/architecture.md`

## Phase 2 实战练习

1. 选一个开源项目，用 Claude 在 15 分钟内画出架构图
2. 为"用户头像上传"功能做完整需求分析和技术方案
3. 使用 Plan Mode 规划一个涉及 5+ 文件的重构任务
4. 让 Claude 写一个 ADR，记录某个技术选型
5. 练习"反向工程"：给一段代码，让 Claude 推断原始需求
